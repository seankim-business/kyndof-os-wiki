# 협업 프로세스 (Collaboration Process)

## 내부 협업

### 팀 내 협업
**일일 동기화**:
- 시간: [시간]
- 형식: Standup / Async
- 내용: 어제/오늘/이슈

**주간 리뷰**:
- 시간: [요일 시간]
- 형식: 회의
- 내용: 진행/계획/회고

### 팀 간 협업
**정기 미팅**:
- 빈도: 주간/월간
- 참석: 팀 리드들
- 목적: 조율 및 의존성 관리

**프로젝트 기반 협업**:
- 크로스-펑셔널 팀 구성
- 명확한 역할 정의 (RACI)
- 정기 체크인

## 외부 협업

### 파트너사 협업
**정기 리뷰**:
- 빈도: 월간/분기별
- 참석: 양사 담당자
- 내용: 진행/이슈/계획

**커뮤니케이션 채널**:
- 주요 연락: [도구]
- 긴급 연락: [방법]
- 문서 공유: [플랫폼]

### 클라이언트 협업
**킥오프**:
1. 프로젝트 목표 정렬
2. 역할 및 책임 정의
3. 커뮤니케이션 계획
4. 마일스톤 합의

**정기 업데이트**:
- 빈도: 주간/격주
- 형식: 리포트 + 미팅
- 내용: 진행/리스크/다음 단계

## 커뮤니케이션 가이드

### 동기 vs 비동기
**동기 (실시간)**:
- 긴급 이슈
- 복잡한 논의
- 브레인스토밍
- 의사결정

**비동기**:
- 상태 업데이트
- 문서 리뷰
- 단순 질의응답
- 계획 공유

### 커뮤니케이션 채널
| 목적 | 채널 | 응답 시간 |
|------|------|---------|
| 긴급 | [도구] | 즉시 |
| 업무 논의 | [도구] | 4시간 |
| 일반 정보 | [도구] | 24시간 |
| 문서화 | [도구] | 비동기 |

## 협업 도구

### 프로젝트 관리
- [도구명]: 작업 추적
- [도구명]: 일정 관리

### 커뮤니케이션
- [도구명]: 메시징
- [도구명]: 화상 회의

### 문서 협업
- [도구명]: 실시간 편집
- [도구명]: 버전 관리

## 협업 원칙

### 1. 투명성 (Transparency)
- 정보 공개 원칙
- 진행 상황 가시성
- 이슈 조기 공유

### 2. 책임성 (Accountability)
- 명확한 역할 정의
- 약속 이행
- 결과 추적

### 3. 존중 (Respect)
- 다양한 의견 수용
- 건설적 피드백
- 시간 존중

### 4. 효율성 (Efficiency)
- 목적 있는 미팅
- 간결한 커뮤니케이션
- 중복 작업 최소화

## 충돌 해결

### 1단계: 조기 식별
- 이슈 감지
- 맥락 파악
- 당사자 확인

### 2단계: 직접 대화
- 1:1 논의
- 사실 확인
- 해결책 모색

### 3단계: 중재
- 제3자 개입
- 객관적 분석
- 합의 도출

### 4단계: 에스컬레이션
- 상위 관리자 개입
- 정책/프로세스 검토
- 최종 결정

## 피드백 문화

### 주는 피드백
- **구체적**: 상황과 행동 명시
- **시의적절**: 즉시 전달
- **건설적**: 개선 방안 제시
- **균형적**: 긍정+개선 병행

### 받는 피드백
- **열린 자세**: 방어 없이 경청
- **질문**: 이해를 위한 확인
- **감사**: 피드백 인정
- **실행**: 개선 적용

## 회의 효율화

### 회의 전
- [ ] 명확한 목적
- [ ] 사전 자료 공유
- [ ] 참석자 최소화
- [ ] 시간 제한 설정

### 회의 중
- [ ] 안건 준수
- [ ] 액션 아이템 기록
- [ ] 결정 사항 확인
- [ ] 시간 관리

### 회의 후
- [ ] 회의록 공유
- [ ] 액션 아이템 할당
- [ ] 후속 조치 추적
