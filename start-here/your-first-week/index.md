# 첫 주 가이드

첫 주는 기술을 배우는 시간이 아닙니다.

**회사를 이해하고, 팀을 만나고, 질문하는 시간입니다.**

---

## 첫 주 목표

### 1주차가 끝나면 당신은:

- [ ] 킨도프의 비전과 전략을 이해함
- [ ] 팀원들과 최소 1번씩 대화함
- [ ] 업무에 필요한 도구들을 셋업함
- [ ] 첫 번째 작은 기여를 완료함
- [ ] 막힐 때 누구에게 물어야 하는지 앎

**완벽하게 일할 필요 없습니다.**

**질문하고, 배우고, 관계를 만드는 것이 목표입니다.**

---

## 일정 가이드

### [Day 1: 환영과 셋업](./day-1/)

**오전:**
- 환영 미팅 (매니저 또는 온보딩 담당자)
- 계정 생성 (이메일, Slack, Notion, GitHub)
- 자리 배정 및 장비 수령

**오후:**
- Wiki 읽기 ([우리는 누구인가](../who-we-are/) 섹션)
- 팀 소개 미팅
- 첫 Slack 메시지 보내기 (#general에 자기소개)

**끝나기 전에:**
- [ ] 모든 계정 로그인 확인
- [ ] 내일 일정 확인
- [ ] 궁금한 점 메모

### Day 2-3: 이해하기

**주요 활동:**
- 프로젝트/제품 데모 보기
- 관련 팀원들과 1:1 미팅 (15-30분씩)
- 업무 관련 문서 읽기
- 작은 task 하나 받아서 시도해보기

**만나야 할 사람들:**
- 직속 매니저 (다시)
- 같은 팀 동료들
- 협업할 다른 팀 담당자
- CTO 또는 CEO (가능하면)

**물어볼 질문들:**
- "우리 팀의 가장 큰 목표는 뭔가요?"
- "제가 가장 먼저 배워야 할 것은 뭔가요?"
- "막히면 누구에게 물어보면 되나요?"

### Day 4-5: 기여하기

**주요 활동:**
- 첫 번째 작은 task 완료하기
- 팀 미팅 참여
- 프로세스 따라해보기 (배포, 테스트, 리뷰 등)

**목표:**
- 작지만 실제로 쓰이는 무언가를 만들기
- 실수해도 괜찮음 (오히려 좋음, 배울 수 있으니까)

## 첫 주 체크리스트

### 계정 & 도구

- [ ] 회사 이메일 활성화
- [ ] Slack 가입 및 채널 참여
- [ ] Notion 초대 수락
- [ ] GitHub 조직 초대 수락
- [ ] (필요시) 추가 도구 (Figma, Railway, etc.)

### 이해

- [ ] [우리는 누구인가](../who-we-are/) 섹션 완독
- [ ] 우리 팀의 OKR 확인
- [ ] 내 업무가 전체 목표에 어떻게 연결되는지 이해
- [ ] 주요 프로세스 파악 (배포, 회의, 리뷰 등)

### 관계

- [ ] 팀원 전원과 최소 1번씩 대화
- [ ] 매니저와 1:1 미팅 (주 1회 예정 확인)
- [ ] 다른 팀 주요 담당자 파악
- [ ] Slack에서 자기소개 완료

### 기여

- [ ] 첫 번째 작은 task 완료
- [ ] 코드/문서 리뷰 요청해보기
- [ ] 팀 미팅에서 최소 1번 발언
- [ ] 개선 아이디어 1개 이상 제안

## 자주 묻는 질문

### "너무 느린 것 같아요. 괜찮나요?"

괜찮습니다. 첫 주는 속도가 목표가 아닙니다.

**제대로 이해하는 것이 목표입니다.**

처음 1주일 천천히 가는 게, 나중에 3개월 헤매는 것보다 훨씬 낫습니다.

### "뭘 물어봐야 할지 모르겠어요."

그것도 괜찮습니다. 다음 질문부터 시작하세요:

1. "우리 팀의 가장 큰 목표가 뭔가요?"
2. "제가 가장 먼저 배워야 할 게 뭔가요?"
3. "이 회사에서 성공하려면 뭘 잘해야 하나요?"

### "실수할까 봐 무서워요."

실수는 배우는 과정입니다.

킨도프는 **"빠르게 실패하고, 빠르게 배우고, 빠르게 공유한다"**를 지향합니다.

실수 자체가 문제가 아닙니다. **같은 실수를 반복하거나, 실수를 숨기는 게 문제입니다.**

### "첫 주에 뭘 만들어야 하나요?"

거창한 걸 만들 필요 없습니다.

- 문서 오타 고치기
- 작은 버그 수정
- 테스트 케이스 하나 추가
- 내부 도구 개선 아이디어 제안

**"작지만 실제로 쓰이는 것"을 목표로 하세요.**

## 도움이 필요하면

### Slack 채널

- `#it-test` - 기술 질문 및 테스트

### 사람

- **매니저** - 업무, 목표, 방향
- **팀 동료** - 기술, 프로세스, 일상
- **온보딩 담당자** - 계정, 도구, 전반적 질문

**주저하지 말고 물어보세요.**

"바보 같은 질문"은 없습니다. 모든 질문은 배우려는 의지입니다.
