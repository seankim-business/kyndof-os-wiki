# Slack

{: .highlight }
> 킨도프의 실시간 소통 허브. 빠른 질문, 협업, 공지가 모두 여기서 이루어집니다.

---

## 왜 Slack인가

예전 회사에서 카카오톡으로 업무 연락을 받아본 적 있나요? 퇴근 후에도 알림이 울리고, 대화가 섞이고, 나중에 찾으려면 스크롤을 한참 해야 했을 겁니다.

킨도프는 업무와 개인을 분리하기 위해 Slack을 선택했습니다.

**Slack의 장점:**
- **검색 가능한 히스토리** - 3개월 전 대화도 바로 찾음
- **채널 기반 정리** - 주제별로 대화가 분리됨
- **스레드 기능** - 하나의 주제를 깔끔하게 정리
- **업무/개인 분리** - 퇴근하면 알림을 끄면 됨
- **봇 연동** - 자동화와 통합 (GitHub, Notion 알림 등)

---

## 시작하기

### 1. 초대 수락

첫날 회사 이메일로 Slack 초대가 옵니다.

1. 이메일에서 "Join Now" 클릭
2. 회사 이메일로 가입
3. 프로필 사진 설정 (얼굴 사진 권장)
4. 이름 형식: `홍길동 (마케팅)` 또는 `Gildong Hong (Marketing)`

### 2. 앱 설치

**데스크톱 앱 권장** - 웹보다 알림이 안정적입니다.

- [macOS 다운로드](https://slack.com/downloads/mac)
- [Windows 다운로드](https://slack.com/downloads/windows)
- 모바일 앱도 설치해두면 좋습니다 (급할 때)

### 3. 알림 설정

**처음엔 기본 설정으로 시작하고, 일주일 후 조정하세요.**

추천 설정:
- `Preferences > Notifications`
- DM, 멘션만 알림 받기
- 방해 금지 시간 설정 (예: 오후 7시 ~ 오전 9시)

## 주요 채널

킨도프 Slack은 목적별로 채널이 나뉘어 있습니다.

### 전사 채널

| 채널 | 용도 |
|------|------|
| `#general` | 전체 공지, 자기소개, 축하 |
| `#random` | 업무 외 잡담, 밈, 추천 |
| `#questions` | 뭐든 물어보는 곳 |

### 팀별 채널

- `#brand-ops` - Brand Ops 팀
- `#atelier` - Atelier 팀
- `#archives` - Archives 팀
- 자신의 팀 채널에 참여하세요

### 기능별 채널

- `#it-test` - 기술 질문, 봇 테스트
- `#design` - 디자인 피드백, 공유
- `#announcements` - 중요 공지 (읽기 전용)

**첫날 할 일:**
1. `#general`에 자기소개 올리기
2. 자신의 팀 채널 참여
3. `#questions` 채널 참여

## 사용 방법

### 메시지 보내기

**채널에 메시지:**
```
채널 선택 → 메시지 입력 → Enter
```

**특정 사람 멘션:**
```
@이름 질문 내용
```

**긴급하지 않은 전체 알림:**
```
@here 공지 내용 (현재 온라인인 사람만)
```

### 스레드 사용하기

**스레드는 Slack의 핵심 기능입니다.**

하나의 주제에 대한 대화는 스레드로 묶으세요:
1. 원본 메시지에 마우스 오버
2. "Reply in thread" 클릭
3. 스레드 내에서 대화

**왜 중요한가:**
- 채널이 깨끗하게 유지됨
- 나중에 대화를 찾기 쉬움
- 관련 없는 사람은 알림 안 받음

### 반응 (이모지)

간단한 응답은 이모지로 충분합니다:

| 이모지 | 의미 |
|--------|------|
| 👍 | 확인했어요 / 동의 |
| ✅ | 완료 |
| 👀 | 보고 있어요 |
| 🙏 | 감사합니다 |
| ⏳ | 진행 중 |

**예시:**
- "회의 10분 연기합니다" → 👍 반응
- "리뷰 부탁드려요" → 👀 반응 후 리뷰

### 검색하기

`Cmd+K` (Mac) 또는 `Ctrl+K` (Windows)

**검색 팁:**
- `from:@이름` - 특정 사람 메시지
- `in:#채널` - 특정 채널에서
- `before:2024-01-01` - 특정 날짜 이전
- 따옴표로 정확한 문구 검색: `"배포 완료"`

## 팁과 에티켓

### DO (이렇게 하세요)

**질문할 때 맥락을 충분히:**
```
❌ "안녕하세요, 질문 있는데요"
   (상대방은 기다려야 함)

✅ "안녕하세요! Notion에서 프로젝트 템플릿 찾고 있는데,
   'Projects' 페이지에서 찾으면 되나요?"
   (바로 답변 가능)
```

**급하지 않으면 채널에:**
```
DM보다 채널에 질문하세요.
다른 팀원도 배울 수 있고, 나중에 검색도 됩니다.
```

**상태 업데이트:**
```
점심, 미팅, 외출 시 상태 변경:
프로필 클릭 → Status 설정
```

### DON'T (이렇게 하지 마세요)

**불필요한 @channel 사용:**
```
❌ @channel 점심 뭐 드실래요?
   (100명에게 알림 감)

✅ @here 점심 같이 드실 분?
   (온라인인 사람만)
```

**"안녕하세요"만 보내기:**
```
❌ "안녕하세요"
   (상대방: "네?" → 기다림 → 비효율)

✅ "안녕하세요! 내일 미팅 시간 변경 가능할까요?"
   (바로 논의 시작)
```

**길고 복잡한 논의:**
```
Slack에서 5개 이상 왕복 → 미팅 잡거나 Notion 문서로
```

## 고급 기능

### 리마인더 설정

```
/remind me to 리뷰 마감 tomorrow at 10am
/remind #채널 주간 미팅 every Monday at 9am
```

### 북마크

중요한 메시지는 북마크:
1. 메시지 옆 "..." 클릭
2. "Add to saved items"
3. `Saved items`에서 확인

### 상태 자동화

```
캘린더 연동 → 미팅 중 자동으로 상태 변경
```

## 자주 묻는 질문

### "알림이 너무 많아요"

채널별로 알림 설정 조정:
1. 채널 이름 클릭
2. "Change notifications"
3. "All messages" → "Mentions & DMs only"

### "DM이 나을까요, 채널이 나을까요?"

| 상황 | 권장 |
|------|------|
| 개인적인 피드백 | DM |
| 다른 사람도 알면 좋은 정보 | 채널 |
| 빠른 1:1 질문 | DM |
| 프로젝트 관련 논의 | 채널 또는 스레드 |

### "퇴근 후에도 알림이 울려요"

`Preferences > Notifications > Do Not Disturb`에서 시간 설정하세요.
