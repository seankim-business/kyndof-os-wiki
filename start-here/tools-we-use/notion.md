# Notion

{: .highlight }
> 킨도프의 두뇌. 모든 문서, 프로젝트, 지식이 이곳에 모입니다.

---

## 왜 Notion인가

Google Docs는 훌륭합니다. 하지만 파일이 100개, 1000개가 되면 어떨까요? 폴더를 만들고, 또 폴더를 만들고, 결국 "문서 어디 있지?"를 반복하게 됩니다.

킨도프는 문서, 데이터베이스, 위키를 하나로 통합하기 위해 Notion을 선택했습니다.

**Notion의 장점:**
- **모든 것이 연결됨** - 문서 안에서 다른 문서 링크
- **데이터베이스** - 프로젝트, 태스크를 표로 관리
- **위키 구조** - 계층적으로 지식 축적
- **템플릿** - 반복 작업 자동화
- **실시간 협업** - 여러 명이 동시 편집

---

## 시작하기

### 1. 초대 수락

첫날 회사 이메일로 Notion 워크스페이스 초대가 옵니다.

1. 이메일에서 "Join Workspace" 클릭
2. 회사 이메일로 로그인 (또는 새 계정 생성)
3. "Kyndof" 워크스페이스 확인

### 2. 앱 설치

**데스크톱 앱 권장** - 더 빠르고 안정적입니다.

- [macOS 다운로드](https://www.notion.so/desktop)
- [Windows 다운로드](https://www.notion.so/desktop)

### 3. 기본 탐색

**사이드바** = 왼쪽 영역
- 워크스페이스 구조가 트리로 표시됨
- 페이지 클릭해서 이동
- 드래그로 구조 변경 (권한 필요)

**빠른 검색** = `Cmd+K` (Mac) 또는 `Ctrl+K` (Windows)
- 페이지 제목, 내용 검색
- 최근 페이지 빠르게 접근

## 워크스페이스 구조

킨도프 Notion은 목적별로 영역이 나뉘어 있습니다.

### 메인 영역

| 영역 | 내용 |
|------|------|
| **Company** | 회사 전체 문서 (비전, 전략, OKR) |
| **Teams** | 팀별 워크스페이스 |
| **Projects** | 진행 중인 프로젝트 |
| **Knowledge** | 가이드, 매뉴얼, SOP |

### 첫날 봐야 할 페이지

1. **Company > Vision & Mission** - 회사 방향 이해
2. **Teams > [내 팀]** - 팀 워크스페이스 확인
3. **Knowledge > Onboarding** - 온보딩 가이드

## 핵심 개념

### 페이지와 블록

Notion에서 모든 것은 **블록**입니다.

- 텍스트 한 줄 = 블록
- 이미지 하나 = 블록
- 테이블 하나 = 블록
- 페이지 하나 = 블록 (안에 더 많은 블록)

**페이지 안에 페이지를 만들 수 있습니다.** 이것이 Notion의 핵심입니다.

```
Company/
├── Vision & Mission
├── OKRs/
│   ├── 2024 Q1
│   ├── 2024 Q2
│   └── ...
└── Strategy/
    ├── 3-Year Plan
    └── Annual Goals
```

### 데이터베이스

Notion의 가장 강력한 기능입니다.

**데이터베이스 = 스마트한 표**

| 항목 | Status | Owner | Due Date |
|------|--------|-------|----------|
| 랜딩 페이지 | 진행중 | 김지민 | 1/15 |
| 이메일 템플릿 | 완료 | 박철수 | 1/10 |

**일반 표와 다른 점:**
- 각 행을 클릭하면 상세 페이지가 열림
- 필터, 정렬, 그룹핑 가능
- 다양한 뷰 (테이블, 보드, 캘린더, 갤러리)

### 링크와 멘션

**페이지 링크:**
- `@페이지이름` 입력 → 페이지 링크
- `[[` 입력 → 페이지 검색해서 링크

**사람 멘션:**
- `@이름` 입력 → 사람에게 알림 전송

**날짜 멘션:**
- `@today`, `@tomorrow`, `@next week` 등

## 사용 방법

### 새 페이지 만들기

1. 사이드바에서 원하는 위치 찾기
2. `+` 버튼 또는 `Cmd+N`
3. 제목 입력
4. 내용 작성

**템플릿 사용:**
- 페이지 생성 시 "Templates" 선택
- 회의록, 프로젝트 브리프 등 팀 템플릿 활용

### 블록 다루기

**새 블록 만들기:**
- Enter = 새 텍스트 블록
- `/` = 블록 메뉴 (heading, bullet, table 등)

**자주 쓰는 `/` 명령:**
| 명령 | 결과 |
|------|------|
| `/h1`, `/h2`, `/h3` | 제목 |
| `/bullet` | 불릿 리스트 |
| `/todo` | 체크박스 |
| `/table` | 테이블 |
| `/quote` | 인용 |
| `/callout` | 강조 박스 |
| `/code` | 코드 블록 |

**블록 이동:**
- 블록 왼쪽 `⋮⋮` 드래그
- 다른 페이지로도 이동 가능

### 검색하기

`Cmd+K` = 만능 검색

**검색 팁:**
- 페이지 제목 일부만 입력해도 됨
- 페이지 내용도 검색됨
- 최근 페이지가 상단에

## 팁과 에티켓

### DO (이렇게 하세요)

**템플릿 적극 활용:**
```
회의록 → 회의록 템플릿 사용
프로젝트 → 프로젝트 브리프 템플릿 사용

일관된 형식 = 나중에 찾기 쉬움
```

**링크로 연결:**
```
관련 문서가 있으면 반드시 링크
"자세한 내용은 @Project Brief 참고"

고아 페이지(어디서도 링크되지 않은 페이지)는 피하기
```

**자주 보는 페이지 즐겨찾기:**
```
페이지 제목 옆 ☆ 클릭
사이드바 상단에서 빠르게 접근
```

### DON'T (이렇게 하지 마세요)

**개인 메모는 개인 섹션에:**
```
❌ 회사 공간에 "내 할일 목록" 만들기
✅ Personal > My Tasks 에 정리

회사 공간은 팀이 함께 쓰는 공간입니다.
```

**구조 함부로 바꾸지 않기:**
```
페이지를 다른 위치로 옮기면 기존 링크가 깨질 수 있습니다.
구조 변경이 필요하면 팀에 먼저 확인하세요.
```

## 주요 워크플로우

### 회의록 작성

1. Team 페이지 > Meetings 데이터베이스
2. "New" 클릭
3. 회의록 템플릿 선택
4. 참석자, 안건, 결정사항 기록
5. 액션 아이템은 담당자 @ 멘션

### 프로젝트 시작

1. Projects 데이터베이스
2. "New" 클릭
3. 프로젝트 브리프 템플릿 선택
4. 목표, 범위, 일정 정의
5. 관련 팀원 @ 멘션으로 공유

### 지식 문서화

1. Knowledge 섹션에 적절한 위치 찾기
2. 새 페이지 생성
3. 가이드/SOP 템플릿 활용
4. 관련 페이지 링크 추가
5. 팀 채널에 공유

## 자주 묻는 질문

### "페이지를 어디에 만들어야 할지 모르겠어요"

| 문서 종류 | 위치 |
|----------|------|
| 회의록 | Teams > [팀] > Meetings |
| 프로젝트 문서 | Projects > [프로젝트] |
| 가이드/매뉴얼 | Knowledge > [카테고리] |
| 개인 메모 | Personal |

모르겠으면 팀원에게 물어보세요.

### "다른 사람이 내 문서를 수정했어요"

Notion은 실시간 협업 도구입니다. 누구나 편집할 수 있습니다.

걱정되면:
1. 페이지 오른쪽 상단 "..." 클릭
2. "Page history" 클릭
3. 이전 버전 확인 및 복원 가능

### "모바일에서도 써야 하나요?"

필수는 아닙니다. 하지만 이동 중 빠른 확인에 유용합니다.
