# CEO Office와 협업하는 방법

{: .highlight }
> 경영진과 효과적으로 소통하고 의사결정을 받는 방법입니다.

## 개요

CEO Office와의 협업은 다른 팀과의 협업과 다릅니다. 일상적인 업무 요청보다는 **전략적 방향 확인**, **중요 의사결정 승인**, **에스컬레이션**이 주요 접점입니다.

이 문서는 언제, 어떻게 CEO Office에 접근해야 하는지, 그리고 효과적인 커뮤니케이션을 위해 무엇을 준비해야 하는지 안내합니다.

## 언제 CEO Office에 접근해야 하나요?

### CEO에게 직접 필요한 경우

1. **전략적 의사결정**: 회사 방향에 영향을 주는 중요한 결정
2. **대규모 투자**: 일정 금액 이상의 지출 승인
3. **외부 관계**: 투자자, 주요 파트너, 미디어 관련
4. **위기 상황**: 법적, 평판, 안전 관련 긴급 이슈
5. **인재**: C-level 또는 핵심 리더 관련

### CTO에게 필요한 경우

1. **운영 이슈**: 재무, HR, IT, 법무 관련
2. **프로세스**: 전사 프로세스 변경이나 개선
3. **시스템**: 새로운 도구나 시스템 도입
4. **팀 간 조율**: 운영적 관점의 팀 간 이슈

### 팀 리더 레벨에서 해결할 것

1. 일상적인 운영 결정
2. 팀 내 인사 이슈
3. 승인된 예산 내 지출
4. 기존 프로세스 내 업무

## 접근 방식

### 정기 채널

| 채널 | 대상 | 목적 |
|------|------|------|
| 주간 Leadership Sync | 팀 리더 → CEO/CTO | 현황 공유, 이슈 논의 |
| 1:1 미팅 | 리더 ↔ CEO/CTO | 심층 논의, 코칭 |
| 월간 All-Hands | 전사 ↔ CEO | 전사 커뮤니케이션 |
| Slack #leadership | 리더들 | 빠른 논의, 공유 |

### Ad-hoc 접근

**긴급하지 않은 경우**
1. Slack DM으로 "상의할 것이 있습니다" 사전 알림
2. 시간 잡아서 미팅 (15-30분)
3. 또는 다음 정기 1:1에서 논의

**긴급한 경우**
1. Slack DM 또는 전화로 즉시 연락
2. 상황 간략히 설명
3. 필요한 의사결정이나 지원 요청

## 효과적인 커뮤니케이션

### 의사결정 요청 시

CEO/CTO의 시간은 한정되어 있습니다. 효율적인 의사결정을 위해:

**준비할 것**
1. **배경**: 왜 이 결정이 필요한가? (1-2문장)
2. **옵션**: 선택지들과 각각의 장단점
3. **추천**: 당신의 추천과 이유
4. **리스크**: 무엇이 잘못될 수 있는가?
5. **요청**: 정확히 무엇을 결정해달라는 것인가?

**형식 (written)**
```
[결정 요청] OOO 건

배경: (왜 필요한지)

옵션:
A. ... (장점/단점)
B. ... (장점/단점)

추천: B (이유)

리스크: ...

요청: B 진행 승인
```

### 보고 시

**좋은 보고의 원칙**
- **결론 먼저**: "좋은 소식/나쁜 소식부터 말씀드리면..."
- **간결하게**: 핵심만, 상세는 질문 시
- **솔직하게**: 좋지 않은 뉴스도 빠르게
- **액션 포함**: "다음 단계로 ... 하겠습니다"

**피해야 할 것**
- 배경 설명만 장황하게 하고 결론이 없는 것
- 문제만 가져오고 솔루션이 없는 것
- 좋지 않은 소식을 숨기거나 늦추는 것

## CEO/CTO의 업무 스타일

### CEO (선빈)

- **선호하는 소통**: 직접적이고 솔직한 대화
- **의사결정 스타일**: 데이터와 직관의 균형
- **피드백 스타일**: 명확한 방향 제시
- **집중 시간**: (특정 시간대가 있다면 명시)

### CTO (지영)

- **선호하는 소통**: 체계적이고 문서화된 커뮤니케이션
- **의사결정 스타일**: 분석적, 시스템적 접근
- **피드백 스타일**: 구조적 피드백
- **집중 시간**: (특정 시간대가 있다면 명시)

## 에스컬레이션 가이드

### 에스컬레이션 전 확인

1. 팀 내에서 해결 시도했는가?
2. 직속 리더와 상의했는가?
3. 관련 팀과 협의했는가?

위 단계를 거쳤는데도 해결되지 않는 경우 에스컬레이션합니다.

### 에스컬레이션 시 포함할 내용

1. **상황**: 무슨 일이 일어났는가
2. **시도**: 무엇을 시도했는가
3. **장애물**: 왜 해결되지 않는가
4. **요청**: 무엇이 필요한가
5. **긴급도**: 얼마나 급한가

### 에스컬레이션 후

- CEO/CTO의 결정을 존중하고 따릅니다
- 결과를 관련자들에게 공유합니다
- 교훈을 프로세스 개선에 반영합니다

## 피드백 받기

CEO/CTO로부터 피드백을 받을 때:

**열린 자세**
- 방어적이지 않게
- 질문을 통해 명확히 이해
- 액션 아이템을 확인

**팔로업**
- 피드백을 반영한 액션
- 진행 상황 업데이트
- 결과 보고

## 하지 말아야 할 것

1. **직속 리더를 건너뛰고 CEO에게 가기**: 에스컬레이션 경로를 지킵니다
2. **결정을 받고 나서 불복하기**: 논의는 결정 전에
3. **나쁜 소식 숨기기**: 일찍 알수록 대응 옵션이 많습니다
4. **자주 에스컬레이션하기**: 팀 레벨에서 해결할 수 있는 것은 팀에서
5. **준비 없이 미팅 요청하기**: 시간 낭비는 신뢰 손상
